Graceful Degradation

Reduced functionality for users of older browsers
Complete refusal in support of old browsers

Graceful Degradation

Pros +
  • Easy
  • Fast
Cons -
  • Support just modern technologies
  • Loss of users --> clients

Progressive Enhancement

Basic functional for all users

Progressive Enhancement

Pros +
  • Stability
  • Site is accessible for everybody
Cons -
  • More work hours

Mobile First!

Prototype and design from mobile version
Just then for wider screens

When to use such method?

If we have a lot of users from mobile devices

Markup types

  • Fixed layout
  • Rubber layout
  • Adaptive layout
  • Responsive layout
Difference

Fixed layout

Block sizes determined in advance.
Layout of blocks in pixels.

Fixed layout

Pros +
  • One version of the layout is enough
  • Predictable layout behavior
  • Easy
Cons -
  • Small screens - horizontal scroll

Rubber layout

The width of the blocks depends on the size of the browser window.
Layout of blocks in percent.

Rubber layout

Pros +
  • Full screen site
Cons -
  • Unpredictability
  • Long strings are possible

Adaptive layout

The site adjusts to the user's device.
Can display content in different ways.
Use CSS @media queries.

Example

Adaptive layout

Pros +
  • Display depends on device size
  • You can optionally hide content
Cons -
  • Layout for each type of device
  • Development takes more time

Responsive layout

Combining approaches of rubber and adaptive layout.
Blocks depend on window size.
Use CSS @media queries.

Example

@media queries

If the width of window less than 650 pixels

@media screen and (max-width: 650px) { }

Responsive layout

Pros +
  • The site looks good everywhere
  • Smoothness for resizing the window
Cons -
  • Layout for each type of device
  • Development takes more time and skills

CSS property position Static

position: static;
  • Element without special positioning
  • Standard default behavior of all elements

CSS property position Relative

position: relative;
  • New properties available: top, bottom, left, right
  • The element moves relative to its location.

CSS property position Absolute

position: absolute;
  • Properties available: top, bottom, left, right
  • The element is relative to the nearest positioned parent
  • The element does not occupy a place in the general flow
  • Element width depends on content

CSS property position Fixed

position: fixed;
  • Properties available: top, bottom, left, right
  • Binds to window coordinates
  • When scrolling, the element remains in the same place
